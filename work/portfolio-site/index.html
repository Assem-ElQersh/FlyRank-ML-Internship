Content Opportunity Scoring Pipeline

FlyRank Machine Learning Internship

Problem: Identifying which content on a client's website is decaying and needs an editorial refresh, without relying on black-box algorithms.

Action: Engineered a robust ML pipeline. Started with a strict data contract and signal audit against 10M+ rows of Search Console data using DuckDB. Developed a transparent baseline action score (Stale * Visible * Recent Volume). Then, trained a constrained Random Forest (max_depth=5) using GroupShuffleSplit to predict traffic decline probability while preventing domain leakage.

Outcome: Discovered that an honest, hard-coded business rule actually beat the generic ML model at predicting the extreme top-50 decaying pages (64.0% vs 58.0% Precision@50)—saving engineering time and proving business value over complex, opaque models.
